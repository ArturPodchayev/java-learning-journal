## что я понял

### == vs equals()

- `==` сравнивает **ссылки** на объект в памяти, а не содержимое
- примитивы (`int`, `boolean`) хранят само значение → `==` работает как ожидаешь
- объекты (`String`, `Person`) хранят ссылку → `==` сравнивает адреса в памяти, не значения
- поэтому два объекта с одинаковыми полями через `==` дадут `false`

### equals() по умолчанию

- все классы в java наследуются от `Object`
- в `Object` уже есть `equals()` — но он сравнивает ссылки, как `==`
- поэтому `p1.equals(p2)` без переопределения тоже даёт `false`

### переопределение equals()

- `@Override` говорит java: "не используй метод из Object, используй мой"
- внутри проверяем три вещи:
  1. это тот же объект? (`this == o`) → сразу `true`
  2. `o` не null и того же класса? → если нет, `false`
  3. поля одинаковые? → сравниваем `age` через `==`, `name` через `Objects.equals()`

```java
@Override
public boolean equals(Object o) {
    if (this == o) return true;
    if (o == null || getClass() != o.getClass()) return false;
    Person person = (Person) o;
    return age == person.age && Objects.equals(name, person.name);
}
```

---

## результат

```
p1 == p2: false        // разные ссылки в памяти
p1.equals(p2): true    // одинаковые поля → true после переопределения
```

---

## вывод

`equals()` нужно переопределять всегда когда хочешь сравнивать объекты по содержимому, а не по адресу в памяти. строки сравниваем только через `.equals()`, никогда через `==`
